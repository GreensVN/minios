## ğŸ”§ Detailed Build Instructions

### Method 1: Using Makefile (Recommended)

```bash
# Clean build
make clean
make all

# Run
make run
```

### Method 2: Manual Build

If Makefile doesn't work, build manually:

```bash
# Create directories
mkdir -p build output

# Assemble bootloader
nasm -f bin bootloader.asm -o build/bootloader.bin

# Assemble interrupt handlers
nasm -f elf32 interrupts.asm -o build/interrupts.o

# Compile kernel
gcc -m32 -c kernel.c -o build/kernel.o \
    -ffreestanding -fno-pie -O2 -Wall \
    -nostdlib -nostdinc -fno-builtin -fno-stack-protector

# Link kernel
ld -m elf_i386 -T linker.ld \
    build/kernel.o build/interrupts.o \
    -o build/kernel.bin

# Create disk image (20 MB)
dd if=/dev/zero of=output/minios.img bs=512 count=40960

# Write bootloader (sector 0)
dd if=build/bootloader.bin of=output/minios.img \
    conv=notrunc bs=512 count=1

# Write kernel (starting at sector 2)
dd if=build/kernel.bin of=output/minios.img \
    seek=2 conv=notrunc bs=512

# Run in QEMU
qemu-system-i386 -drive file=output/minios.img,format=raw -m 256M
```

## ğŸ› Troubleshooting

### Problem: Build fails with "command not found"

**Solution:** Install missing dependencies
```bash
# Ubuntu/Debian
sudo apt install build-essential nasm

# Check versions
gcc --version
nasm --version
ld --version
```

### Problem: "Invalid boot signature"

**Solution:** Bootloader assembly is wrong
```bash
# Verify boot signature
hexdump -n 2 -s 510 -e '/1 "%02x"' build/bootloader.bin
# Should output: 55aa
```

### Problem: Kernel won't load in QEMU

**Solution:** Check kernel size
```bash
# Kernel should be < 32 KB for default bootloader
ls -lh build/kernel.bin

# If too large, increase KERNEL_SECTORS in bootloader.asm
```

### Problem: Black screen in QEMU

**Solution:** Try different QEMU options
```bash
# Try without KVM
qemu-system-i386 -drive file=output/minios.img,format=raw \
    -m 256M -no-reboot -no-shutdown

# Try with serial output
qemu-system-i386 -drive file=output/minios.img,format=raw \
    -m 256M -serial stdio
```

### Problem: Won't boot on real hardware

**Possible causes:**
1. **UEFI vs BIOS:** MiniOS uses BIOS. Disable UEFI/Secure Boot in BIOS
2. **USB not bootable:** Try different USB port or drive
3. **Write permissions:** Use `sudo` for dd command
4. **Wrong device:** Double-check device name with `lsblk`

**Solutions:**
```bash
# Enable legacy BIOS mode in UEFI settings
# Disable Secure Boot
# Use USB 2.0 port instead of USB 3.0

# Verify USB was written correctly
sudo dd if=/dev/sdX of=/tmp/test.img bs=512 count=1
hexdump -C /tmp/test.img | head
# Should see bootloader code and 55 AA at end
```

## ğŸ”§ Key Components Explained

### 1. Bootloader (bootloader.asm)
```
What it does:
âœ… Loaded by BIOS at 0x7C00
âœ… Enables A20 gate (access >1MB memory)
âœ… Detects memory
âœ… Loads kernel from sector 2
âœ… Switches to Protected Mode
âœ… Jumps to kernel

Key: Must be exactly 512 bytes, end with 0x55AA
```

### 2. Kernel (kernel.c)
```
What it does:
âœ… VGA text mode (80x25 color text)
âœ… Interrupt handling (IDT/ISR/IRQ)
âœ… Timer (PIT at 100 Hz)
âœ… Keyboard input (PS/2)
âœ… Basic memory management
âœ… Exception handlers (for crashes)

Key: Entry point is kernel_main()
```

### 3. Interrupts (interrupts.asm)
```
What it does:
âœ… Low-level interrupt stubs
âœ… CPU exceptions (0-31)
âœ… Hardware IRQs (32-47)
âœ… Saves/restores registers
âœ… Calls C handlers

Key: Links assembly to C code
```

### 4. Linker Script (linker.ld)
```
What it does:
âœ… Organizes memory layout
âœ… Places code at 0x1000
âœ… Separates sections (text/data/bss)
âœ… Sets entry point

Key: Tells linker where to put everything
```

## ğŸ› Quick Troubleshooting

| Problem | Solution |
|---------|----------|
| **Command not found** | Install dependencies: `sudo apt install nasm gcc qemu-system-x86` |
| **Boot signature error** | Bootloader must end with `dw 0xAA55` |
| **Black screen in QEMU** | Check kernel size < 32KB, or increase KERNEL_SECTORS |

| **Won't boot on hardware** | Disable UEFI/Secure Boot, use legacy BIOS |

| **Compile errors** | Use 32-bit gcc: `gcc -m32`, install `gcc-multilib` |
| **Link errors** | Check linker.ld exists, use `-T linker.ld` |
